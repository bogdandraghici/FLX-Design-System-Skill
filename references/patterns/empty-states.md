The FlowX empty state is what a section or page shows when it has nothing to display yet (status: Proposal). It is a single model: a centered column flex stack, text-align center, in the FlowX font (var(--font-flowx), Open Sans), made of a 24px icon, a bold title, a caption subtitle, and an optional secondary-button call to action, optionally wrapped in a bordered container. The empty state is placed centered inside whatever section has no content, which is usually a Primary card (white surface, the 2px 2px 24px rgba(22,52,98,.08) shadow, 16px radius, see the Cards pattern) standing in for the page's missing content. It replaces only the content area: keep filters, search, and other actions visible above it, so a table's header row and toolbar stay put while only the row area becomes the empty state.

Anatomy, top to bottom. The icon is a 24px Phosphor icon in neutrals-500 (#64748b) with a 12px bottom margin; match it to the situation, always use the MagnifyingGlass icon for no-results states, and fall back to CircleDashed when no situation-specific icon fits. The title is 14px weight 700 with a 24px line-height in neutrals-900 (#1d232c) and a 4px bottom margin. The subtitle (the caption) is 12px regular with a 16px line-height in neutrals-500; it is the only element that survives in the Small size. The CTA is a white secondary button: inline-flex with a 6px gap, white (#ffffff) surface, 1px solid neutrals-100 (#e3e8ed) border, 8px radius, and 14px weight-600 blue-500 (#006bd8) text; it comes in Medium (36px tall, 6px 12px padding) and Small (32px tall, 6px 8px padding), and may carry an optional trailing icon such as a 16px ArrowRight. It is on by default for No content empty states but not mandatory: only add one when there is a clear next step, and drop it when there is no single clear next step. The optional bordered container wraps the centered stack with 24px padding, a white surface, a 1px solid neutrals-100 border, and a 16px radius, and is used inside a Primary card to frame the empty state; add it when the empty state has no parent container of its own or the layout needs a clear boundary, and skip it when a parent such as a table, panel, or card already frames the space.

There are two sizes. Medium is the full model (icon, title, subtitle, and optional CTA); use it where there is room to guide the user, and use a Medium CTA when the empty state sits in an area with M-sized elements such as admin pages. The documented Medium example shows it inside the row area of a real entity table (the Values Table component's entity mode: 0.5px #cbd1db border, 50px header with 50%-opacity neutrals-50 background) — a CircleDashed icon, the title "There are no stores yet", the subtitle "Add content manually, generate it from workflows or create it automatically when testing", and a Medium "Add Store" CTA, with only the row area holding the empty state while the table header stays visible. Small is subtitle only: it drops the icon, title, and CTA, leaving just the 12px caption. Use Small in cramped or dense sections where only the empty message fits, such as a dropdown panel or a tight subsection; when the surrounding area uses small components such as a config panel, pair a Small CTA with it only if a CTA is genuinely needed. Match the size to the components around the empty state.

The same model covers every no-content situation; three documented scenarios are first use / no data yet (the Classes example: a Stack icon, the title "No classes yet", the subtitle "The categories your model sorts pages into — you need at least two to train.", and a Small "Add Class" CTA with no icon), no results (the workflow node search example: a MagnifyingGlass icon, the title "No results", the subtitle "There are no nodes matching your search.", and no CTA; no-results states always use the MagnifyingGlass icon), and canvas empty state (the UI Designer canvas example: a Layout icon, the title "UI Designer", the subtitle "Drag components from the sidebar", and no CTA or container). Canvas empty states are centered in the view and do not use a bordered container.

Copy guidance. Title: name the state rather than decorate it, use sentence case with no ending period and no more than five words, and follow the templates "No {items} yet" for no-data and "No results" for no-results. Subtitle: add new information and never restate the title, write one sentence in sentence case with an ending period, and by default describe what fills the space (such as "UI Flows you create appear here."); when there is no CTA, explain how to start (such as "Drag components from the sidebar."); when the value isn't obvious, explain why it matters (such as "Metadata keys make content easier to find."); and for no-results states, flip to recovery copy (such as "No nodes match your search."). Button: use verb-first sentence case copy with no ending period, match the toolbar or destination label exactly, and include only one button, only when there is a clear next step. Across the app: write both states for anything searchable, since no-data is not the same as no-results, and use one consistent name per concept across the title, subtitle, and button.

Guidelines: use a centered stack to explain why a section has no content and what action fills it; one model covers all no-content situations, so just match the icon and copy to the situation (no-results states always use the MagnifyingGlass icon); the empty state replaces only the content area, so keep filters, search, and other actions visible above it; the CTA is on by default for No content empty states but not mandatory, so only add one when there is a clear next step; the container is optional, add it when the empty state has no parent container of its own or the layout needs a clear boundary, and skip it when a parent such as a table, panel, or card already frames the space; canvas empty states are centered in the view and skip the container; match the size to the components around the empty state; use Medium (the full model) where there is room to guide the user and a Medium CTA in areas with M-sized elements such as admin pages; use Small (subtitle only) for limited space such as a dropdown panel or a tight subsection and a Small CTA when the surrounding area uses small components such as a config panel; and fall back to the CircleDashed icon when no situation-specific icon fits. Related patterns and components are Loaders, Cards, and Button. Dark-theme values are not documented; the pattern page keeps its example mockups light.
