The FlowX empty state is what a section or page shows when it has nothing to display yet (status: Proposal). It is a single model: a centered column flex stack, text-align center, in the FlowX font (var(--font-flowx), Open Sans), made of a 24px icon, a bold title, a caption subtitle, and an optional secondary-button call to action, optionally wrapped in a bordered container. The empty state is placed centered inside whatever section has no content, which is usually a Primary card (white surface, the 2px 2px 24px rgba(22,52,98,.08) shadow, 16px radius, see the Cards pattern) standing in for the page's missing content.

Anatomy, top to bottom. The icon is a 24px Phosphor icon in neutrals-500 (#64748b) with a 12px bottom margin; match it to the situation and fall back to CircleDashed when no situation-specific icon fits. The title is 14px weight 700 with a 24px line-height in neutrals-900 (#1d232c) and a 4px bottom margin. The subtitle (the caption) is 12px regular with a 16px line-height in neutrals-500; it is the only element that survives in the Small size. The CTA is optional, sits 24px below the subtitle, and is a white secondary button: inline-flex with a 6px gap, white (#ffffff) surface, 1px solid neutrals-100 (#e3e8ed) border, 8px radius, and 14px weight-600 blue-500 (#006bd8) text; it comes in Medium (36px tall, 6px 12px padding) and Small (32px tall, 6px 8px padding), and may carry an optional trailing icon such as a 16px ArrowRight. Only add a CTA when there is a clear next step. The optional bordered container wraps the centered stack with 24px padding, a white surface, a 1px solid neutrals-100 border, and a 16px radius, and is used inside a Primary card to frame the empty state.

There are two sizes. Medium is the full model (icon, title, subtitle, and optional CTA); use it where there is room to guide the user, and use a Medium CTA when the empty state sits in an area with M-sized elements such as admin pages. Small is subtitle only: it drops the icon, title, and CTA, leaving just the 12px caption. Use Small in cramped or dense sections where only the empty message fits, such as a dropdown panel or a tight subsection; when the surrounding area uses small components such as a config panel, pair a Small CTA with it only if a CTA is genuinely needed. Match the size to the components around the empty state.

The same model covers every no-content situation; two documented scenarios are first use / no data yet (the Trained Models example: a Stack icon, the title "No trained models yet", the subtitle "Each training run adds a version here.", and a Small "Go to Classes" CTA with an ArrowRight) and no results (the workflow node search example: a FlowArrow icon, the title "No Results", the subtitle "There are no nodes matching your search.", and no CTA).

Guidelines: use a centered stack to explain why a section has no content and what action fills it; one model covers all no-content situations, so just match the icon and copy to the situation; only add a CTA when there is a clear next step; match the size to the components around the empty state; use Medium (the full model) where there is room to guide the user and a Medium CTA in areas with M-sized elements such as admin pages; use Small (subtitle only) for limited space such as a dropdown panel or a tight subsection and a Small CTA when the surrounding area uses small components such as a config panel; and fall back to the CircleDashed icon when no situation-specific icon fits. Related patterns and components are Loaders, Cards, and Button. Dark-theme values are not documented; the pattern page keeps its example mockups light.
