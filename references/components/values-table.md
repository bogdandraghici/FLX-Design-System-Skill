Tables (formerly called Values Table) is an editable table for managing simple items that don't warrant their own details page, with inline editing and validation. It accepts the following props: editMode (boolean, default false), error (boolean, default false), batchEdit (boolean, default false), and entityMode (boolean, default false). The table has four states: read-only, editing, error, and warning. Its anatomy consists of a Table Container with white background, 1px #cbd1db border, and 16px corner radius. The Header Row has a semi-transparent #f7f8f980 background. Each Header Column contains a 12px SemiBold label in #64748b. Data Rows hold cells separated by vertical dividers with an Action Column. Read-only Cells display 14px Regular text in #0f1114, with placeholder text at 30% opacity. Editable Cells are 28px tall inline inputs with 8px corner radius and 1px border, #cbd1db by default and #006bd8 when focused. Error Cells have a red #e62200 border. A warning icon in #fd6b1c appears on a read-only cell when an external configuration change has made the value invalid; unlike validation errors, warnings are not caused by direct editing. The Action Column contains Edit and Delete buttons in read-only mode (28x28, 8px corner radius, white fill, #e3e8ed border; when disabled they have a transparent fill, #cbd1db border, and #a6b0be icon), and Save and Cancel buttons in editing mode (28x28 each, 8px corner radius). The Save Button uses #008060 green when active and #e3e8ed fill with a #8390a2 icon when disabled. The Cancel Button uses #e62200 red. Error Messages appear below cells in 12px Regular red text. Tooltips show detailed validation messages on hover. Typography uses Open Sans throughout. Header text is 12px SemiBold #64748b. Cell value text is 14px Regular. Error message text is 12px Regular. The container has a 16px corner radius with 1px #cbd1db border. Action buttons are 28x28 with an 8px corner radius. The table comes in a bordered and a standalone variant: use bordered when the table sits inside another container like a card or settings panel, and standalone when the table is the card itself, where it drops the border, uses a white header background, and gets its own shadow of 2px 2px 24px rgba(22,52,98,0.08). batchEdit enables always-on editing where all rows render inline inputs at once; use it when the page already has a save/discard mechanism, and row-by-row editing when users update rows independently. entityMode renders an entity table for complex items that open into a detail page or modal, like workflows or integrations; value tables are for simple items without their own detail page. When the table has many rows, keep the header sticky so column context is visible during scroll. This component has no inverted/dark variant. The container background is #ffffff. The header background is #f7f8f980. Value text color is #0f1114. Error border color is #e62200. Save active color is #008060. Cancel color is #e62200. Edit and Delete button border color is #e3e8ed.
