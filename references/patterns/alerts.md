Alerts is a product-level pattern (not a registered base component): a confirmation dialog for destructive, irreversible actions, and a specialized fixed-size variant of the Modal pattern. It names the resource, explains the consequence, and offers a clearly-marked destructive action beside a safe Cancel. It accepts the following props: title (string, a heading in the form [Action] + [Resource name] such as "Delete process_data"), subtitle (string, the resource type such as "Reusable UI Template"), body (string, a plain-language explanation of the consequence), and showReferences (boolean, default true, toggling the optional reference row). Alerts have no size axis; they are fixed at S / 600px, so there is no Sizes section. The container is a vertical auto-layout surface 600px wide with a 16px corner radius, a drop shadow of 2px 2px 24px rgba(22, 52, 98, 0.12), and 32px padding and a 32px gap between sections. The background is #ffffff (grays/50); only light-mode colors are specified in the sources, and no dark/inverted variant is defined for the alert dialog.

The anatomy from top to bottom: a centered header stack (4px gap, fills width) holding the title and subtitle; a left-aligned body paragraph explaining the consequence; an optional reference row; and a horizontal action bar at the bottom. The title is Open Sans Bold (700) at 24px with a 38px line-height, colored #2a313a (neutrals/800), and follows the [Action] + [Resource name] format. The subtitle is Open Sans Regular (400) at 12px/16px, colored #64748b (neutrals/500), and names the resource type. A 24x24 X close icon is absolutely positioned in the top-right corner (inset by the container padding), is always rendered, and dismisses (cancels) the alert; its fill is #64748b. The body is Open Sans Regular (400) at 14px/24px (fontsize/m), left-aligned and colored #1d232c (neutrals/900), and must state what the action will affect. The optional reference row (toggled via showReferences, default visible) puts "Referenced in N processes." on the left in bold (Open Sans 14px/24px, weight 700, #1d232c) and a "View references" text link on the right (Open Sans SemiBold 600, 14px/24px, #006bd8 / blue-600) with a leading ListMagnifyingGlass icon.

The action bar fills the width, hugs its height, uses a 16px gap between buttons, and is right-aligned (primaryAlign MAX). Buttons have an 8px corner radius and Open Sans SemiBold (600) 14px/24px text. The secondary (Cancel) button is #ffffff with a 1px #e3e8ed (grays/200) stroke and #006bd8 (blue-600) text; the primary destructive button is a red fill #e62200 (red/500) with #ffffff text. Cancel sits on the left and the destructive primary action sits on the right. This is the intentional exception to the general modal rule of never placing a destructive action on the right: an Alert's whole purpose is the destructive confirmation, so the red primary is always the right-most action. Never place the destructive action on the left or style it as the safe choice.

Usage guidance: reserve alerts for destructive, irreversible actions such as deleting or discarding a resource, where the user must consciously confirm; for non-destructive confirmations, use a standard Modal instead. Name the specific resource in the title, explain the consequence (including what may break) in the body, use the red Danger primary button labelled with the destructive verb ("Delete", "Discard"), and keep Cancel as the safe default with dismissal always available via Cancel, the X, or Escape. Don't use an Alert for non-destructive confirmations, don't label the primary action ambiguously ("OK", "Yes"), don't omit the consequence, and don't put the destructive action on the left. Considerations: set focus to Cancel (not the destructive action) when the alert opens so an accidental Enter doesn't confirm; phrase the primary button as the destructive verb; and use the reference row when the resource is referenced elsewhere so users understand the blast radius before confirming. Accessibility: role="alertdialog" so assistive tech announces it as an interruption requiring a response, aria-modal="true", aria-labelledby on the title, and aria-describedby on the body paragraph; trap Tab focus between Cancel, the destructive action, and the close icon, dismiss on Escape (never confirm), and default focus to Cancel. Related components: button, modals.

Values not specified in the sources: any dark/inverted color variant for the alert dialog, and button height/padding (not defined in the Figma spec).
